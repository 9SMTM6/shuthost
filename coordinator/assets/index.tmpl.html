üí° Wake-on-LAN (WOL) Requirements

The agent requires Wake-on-LAN for remote startup functionality:

  • Motherboard Support: Your motherboard must support WOL
  • BIOS Configuration: WOL must be enabled in BIOS/UEFI settings
  • OS Configuration: WOL must be enabled in the operating system
  • Power State Limitation: Some systems only support WOL from sleep mode, not full shutdown
  • Network Requirements: Requires network broadcast support and host reachability
  • VM Considerations: If running in a VM, use bridged networking mode

Note: The installer tests network reachability but won't fail if the test is unsuccessful.

üîí Required Security Configuration

Proxy/Authentication Bypass Required

For the installation scripts to work properly, certain endpoints must be accessible without authentication. If you're using a reverse proxy with authentication (like Authelia, NPM with auth, etc.), you need to configure bypass rules for:

  • /download/* - Installation script downloads
  • /api/m2m/* - Machine-to-machine API communication

This allows the installer scripts to download necessary files and communicate with the coordinator API without requiring user authentication.

Configuration Examples

Authelia:

- domain: your-shuthost-domain.com policy: bypass resources: - '^/download/(.*) - '^/api/m2m/(.*)$'

Nginx Proxy Manager with Authentication:

# In your proxy host's advanced configuration location ~ ^/(download|api/m2m)/ { auth_basic off; proxy_pass http://your-shuthost-backend; }

Traefik with ForwardAuth:

# Add to your service labels - "traefik.http.routers.shuthost-bypass.rule=Host(`your-domain.com`) && PathPrefix(`/download`, `/api/m2m`)" - "traefik.http.routers.shuthost-bypass.priority=100" # Remove auth middleware for bypass routes

Replace domain/backend references with your actual configuration values.
